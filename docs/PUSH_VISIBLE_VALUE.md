# PUSH VISIBLE VALUE DOCTRINE

Every meaningful LOCUS push must leave behind:

- a command
- a demo
- a screenshot
- a benchmark
- a trace
- a landing improvement
- a new visible capability
- or a test the user can feel

Never allow the repository to disappear into invisible refactors for 100 commits.

The machine must continuously reveal its growing power.

---

# GOOD PUSHES

## GOOD

```text
Added:
  locus region-search

Now a tiny crop can be found inside a larger image.
```

## GOOD

```text
Added:
  exact PDF landing

Double-click result now opens exact PDF page.
```

## GOOD

```text
Added:
  continuity trace

LOCUS can now reconstruct screenshot ancestry.
```

---

# BAD PUSHES

## BAD

```text
Refactored utility abstractions.
```

## BAD

```text
Cleaned up architecture.
```

## BAD

```text
Internal improvements.
```

Invisible work is allowed.
Invisible progress is not.

---

# THE COMPOUNDING EFFECT

If every push ships a visible capability:

commit 5:
  "it can find duplicates"

commit 12:
  "it can find crops"

commit 21:
  "it opens exact PDF pages"

commit 37:
  "it reconstructs meme ancestry"

commit 52:
  "it finds comic panels"

commit 73:
  "it traces screenshots across years"

commit 100:
  the demo already exists.

The audience watched it become inevitable.
